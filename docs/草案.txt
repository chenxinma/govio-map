1. 产品概述
1.1 产品定位
一款以无限画布为核心交互界面的数据治理工具，用户通过自然语言指令生成数据查询、处理逻辑，并以可视化卡片形式在画布上编排数据血缘关系。

1.2 核心价值
所见即所得：SQL/DataFrame 逻辑与元数据可视化呈现

可追溯性：画布连线即数据血缘地图

低门槛：Chat 自然语言替代复杂编码

1.3 用户画像
角色	使用场景
数据分析师	快速探查数据、生成报表逻辑
数据工程师	梳理数据血缘、设计ETL流程
业务人员	通过自然语言获取数据洞察

5. 数据流与状态管理
5.1 Zustand Store 结构
typescript
interface CanvasStore {
  // 画布状态
  nodes: CanvasNode[];
  edges: CanvasEdge[];
  viewport: Viewport;
  
  // 选中状态
  selectedNodeIds: string[];
  selectedEdgeIds: string[];
  
  // Chat 上下文
  chatContext: ChatContext;
  
  // 操作历史（用于撤销/重做）
  history: {
    past: CanvasSnapshot[];
    future: CanvasSnapshot[];
  };
  
  // Actions
  addNode: (node: Partial<CanvasNode>) => void;
  updateNode: (id: string, data: Partial<CanvasNodeData>) => void;
  removeNodes: (ids: string[]) => void;
  addEdge: (edge: Partial<CanvasEdge>) => void;
  removeEdges: (ids: string[]) => void;
  
  // 布局 Actions
  autoLayout: (nodeIds?: string[]) => void;
  incrementalLayout: (newNodeId: string) => void;
  
  // 历史 Actions
  undo: () => void;
  redo: () => void;
  
  // Chat Actions
  sendMessage: (content: string, attachments?: Attachment[]) => Promise<void>;
  addNodeToContext: (nodeId: string) => void;
  clearContext: () => void;
}

8. 性能优化策略
8.1 画布渲染优化
策略	实现方式	预期效果
虚拟化节点	React Flow 内置视口裁剪	大量节点时保持 60fps
节点内容懒加载	IntersectionObserver	减少初始渲染开销
Schema 列表虚拟滚动	react-window	字段数量多时不卡顿
防抖排版计算	lodash.debounce	避免频繁重排

9. 错误处理与降级
9.1 错误边界
场景	处理方式
AI 服务超时	显示重试按钮，保留用户输入
SQL 执行失败	在节点上显示错误标记，悬浮展示错误信息
布局计算失败	回退到网格排列
WebSocket 断开	自动重连，期间禁用 Chat 输入
9.2 离线支持
画布状态自动保存至 LocalStorage

恢复时提示用户上次编辑时间

AI 功能离线时自动禁用
